# 自建管线 vs 直接调用大模型 — 对比矩阵

*Last updated: 2026-06-15*

## 对比对象

| 方案 A：自建管线 | 方案 B：直接调用大模型 API |
|---|---|
| rembg → Canny → SD + ControlNet + LoRA | gpt-image-1 / fal.ai Flux |

---

## 核心维度对比

| 维度 | 方案 A：自建管线 | 方案 B：大模型 API |
|---|---|---|
| **风格一致性** | ⭐⭐⭐⭐⭐ LoRA 绑定风格，ControlNet 锁结构，同参数高度一致 | ⭐⭐⭐ Prompt 只能"建议"风格，输入变化大时可能漂移 |
| **ID 保真度** | ⭐⭐⭐⭐⭐ Canny/Lineart 边缘约束，保证"看得出是你" | ⭐⭐~⭐⭐⭐ 依赖模型理解，可能"美化过度"不像本人 |
| **可定制深度** | ⭐⭐⭐⭐⭐ 线条粗细、细节密度、色调全可调；可练人物专属 LoRA | ⭐⭐ 调参空间仅限 prompt、seed；无法 fine-tune |
| **艺术上限** | ⭐⭐⭐~⭐⭐⭐⭐ 受限于 SD 基模 + LoRA 质量 | ⭐⭐⭐⭐⭐ 大规模预训练，单 prompt 可出惊艳效果 |
| **起步成本** | 高：GPU 服务器 + 模型部署 + 管线开发 | 低：零基础设施，几行 API 调用跑通 |
| **边际成本（单张）** | 极低：约 ¥0.01~0.05/张（算力电费） | 中：$0.002~0.04/张（按模型计费） |
| **规模拐点** | 约 500~1000 张/天后自建更便宜 | 低量划算，量越大越贵 |
| **延迟** | 2~10 秒/张 | 3~15 秒/张（API 负载依赖） |
| **并发能力** | 取决于自有 GPU 数量 | 理论无限（有 rate limit，可提额） |
| **部署复杂度** | 高：ComfyUI/WebUI、模型管理、GPU 运维 | 极低：HTTP POST + base64 图片 |
| **迭代速度** | 慢：换风格需重新训练 LoRA（30min~2h） | 快：改 prompt 秒级生效 |
| **隐私安全** | 照片本地处理，数据不出服务器 | 照片上传第三方，有合规风险 |
| **依赖风险** | 可控：开源模型，不会被停服/涨价 | 高：模型可能下架/涨价/内容审查变更 |
| **可复现性** | 强：固定 seed + 固定权重完全可复现 | 弱：模型静默更新后同 prompt+seed 可能变化 |
| **多风格覆盖** | 弱：每种风格需单独练 LoRA 或换工作流 | 强：改 prompt 即切风格 |

## 选型建议

```
                    自建管线                    大模型 API
                    ◄────────────────────────►
                    
越适合自建 ←                    ｜                    → 越适合 API
                    
用户量 < 100/天                 ｜ 用户量 > 1000/天
高度在意风格一致性              ｜ 需要快速验证 PMF
对 ID 保真度有严格要求          ｜ 需要覆盖多种风格
数据隐私是硬约束                ｜ 无隐私合规包袱
有 GPU 运维能力                 ｜ 团队无 ML 背景
长期运营项目                    ｜ 短期/实验性项目
```

## 三阶段演进策略

**第一阶段（验证期，当前）**：直接用 API
- 主力：`gpt-image-1`（images.edit，img2img）
- 对比：`fal-ai/flux/schnell`（速度对比）
- 目标：验证"用户到底要不要线条头像"

**第二阶段（风格定型期）**：API + 风格资产积累
- 用 API 大量生成风格变体，筛选用户最喜欢的 2-3 种
- 用这些图练 LoRA，为自建管线储备风格资产
- 引入用户满意度隐式信号（下载率 = 满意度代理指标）

**第三阶段（规模化）**：切自建管线
- 触发条件：日均 > 500 张，且自建边际成本 < API 成本
- ComfyUI + ControlNet + LoRA 服务"稳定风格"
- 保留 API 作为"新风格探索"通道

## MVP 两个模型的实际对比维度

对比时关注以下指标（人工评估）：

| 指标 | 说明 |
|---|---|
| ID 保真度 | 生成结果"看得出是本人"的比例 |
| 风格一致性 | 不同输入图产出风格的稳定程度 |
| 线条质量 | 线条是否清晰、干净、不模糊 |
| 延迟 | 实测 P50/P95 响应时间 |
| 成本 | 每张实际花费 |
| 拒绝率 | API 触发内容过滤的比例 |
